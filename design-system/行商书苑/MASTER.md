# 行商书苑小程序视觉规范

> 来源：用户提供的 HTML 原型 `002.html`、沉浸式 Navbar demo `docs/demos/immersive-navbar-demo.html`，结合 `ui-ux-pro-max` 对移动端 B2B 商业服务产品的规则进行收敛。构建具体页面时，以本文档为全局视觉规范；页面如需偏离，需在 `design-system/行商书苑/pages/` 下增加覆盖说明。

## 产品气质

行商书苑是面向财税机构与本地服务商的商业撮合、线下活动和生态服务平台。“书苑”是品牌名，不代表产品要做成书籍、阅读或传统文化社区。视觉应体现“专业、可信、资源连接、活动现场感、行业服务”，避免儿童化、游戏化、文化散文式表达和过度 AI 生成感。

关键词：

- 财税服务
- B2B 生态平台
- 商业撮合
- 线下活动
- 本地资源对接
- 专业可信
- 克制科技感
- 深蓝与金色强调
- 高信息密度但保持清晰

## 设计原则

1. 以移动端优先，默认宽度按 390px 小程序视口组织。
2. 首页优先使用沉浸式首屏：顶部导航压在活动现场、商务会议、资源对接等真实背景图上；滚动后切换为白底实色导航。
3. 内容承载以白色卡片为主，卡片之间用 10-16px 间距分组。
4. 金色只用于高价值动作、会员权益、峰会报名、关键数据，不大面积铺满。
5. 减少装饰圆、强渐变、漂浮光效、夸张阴影，避免“AI 味”。
6. 不用 emoji 作为正式 UI 图标；原型中的 emoji 只作为信息架构提示。
7. 数字和标签要服务业务判断，避免堆砌无依据的营销数字。
8. 背景图必须服务业务场景，优先使用活动现场、商务会议、路演、洽谈、签约、城市商务空间等图像；不要使用书架、古风书院、山水文旅等误导业务定位的图片。

## 色彩系统

| Token | Hex | 用途 |
| --- | --- | --- |
| `brand` | `#0A1F5C` | 顶部品牌区、主按钮、选中态 |
| `brand-light` | `#1A3580` | 深蓝渐变辅助、轻强调背景 |
| `brand-deep` | `#050E2E` | 数据带、深色卡片底 |
| `gold` | `#C8960C` | 会员、高价值 CTA、关键数据 |
| `gold-light` | `#F0B429` | 金色高亮、深色背景上的数字 |
| `gold-soft` | `#FFF3D0` | 金色标签浅底 |
| `tech` | `#1677FF` | 链接、次级可点击文本、信息标签 |
| `canvas` | `#F5F7FA` | 页面底色 |
| `line` | `#E8ECF2` | 分割线、边框 |
| `muted` | `#6B7A99` | 次级说明文本 |
| `ink` | `#333D55` | 正文与卡片标题 |
| `danger` | `#E53E3E` | 价格、紧急状态、未读提醒 |
| `success` | `#38A169` | 认证、完成、成功状态 |

背景不要使用暖米色或大面积红色。深蓝与白色是主骨架，金色是节制的业务强调。沉浸式首屏可在背景图上叠加深蓝遮罩，遮罩只用于保证文字与导航可读性，不应把背景处理成纯色或明显渐变海报。

## 排版

字体：

- 小程序端使用系统字体栈：`-apple-system`, `PingFang SC`, `Helvetica Neue`, `sans-serif`
- 不引入 Google Fonts，不使用儿童化字体。

字号：

| 层级 | 建议 |
| --- | --- |
| 页面标题 | 20px / 700 |
| 卡片标题 | 16-18px / 700 |
| 列表标题 | 14px / 600-700 |
| 正文说明 | 13-14px / 400-500 |
| 标签、辅助信息 | 11-12px / 500-600 |
| 关键数字 | 16-18px / 800 |

中文行高保持 1.45-1.65。不要使用负字距，不要用超大标题填满小程序首屏。

## 布局与间距

- 页面左右内边距：16px。
- 小程序顶部区域需要考虑状态栏与微信胶囊按钮，沉浸式首屏建议导航高度约 88-96px。
- 页面顶部品牌区：左右 14-20px，上下 12-24px；如果是沉浸式首屏，内容需要从状态栏下方开始组织。
- 卡片内边距：14-16px。
- 卡片圆角：12px，会员/重点权益卡可用 14px。
- 按钮圆角：8-10px。
- 标签圆角：4-6px，筛选 chip 可用 18px。
- 底部导航高度参考 76px，文字 11px。

## 组件规范

### 沉浸式顶部导航

- 首页和重点活动页可采用沉浸式顶部导航：状态栏、页面标题、城市/业务说明、微信胶囊按钮压在背景图上。
- 初始状态：导航背景透明或轻微深色遮罩，文字白色，说明文字使用 `rgba(255,255,255,0.72)`。
- 滚动状态：导航切换为 `rgba(255,255,255,0.96)` 或白色实底，标题使用 `ink`，说明使用 `muted`，底部可加 `0 2px 16px rgba(10,31,92,0.10)` 阴影。
- 背景图上必须有足够深的业务遮罩，首屏标题和搜索入口对比度要稳定。
- 微信胶囊按钮区域应被保留，不要用自定义按钮占用右上角胶囊位置。
- 导航信息保持简洁：页面标题 + 一行城市/业务说明即可，不放多行营销文案。
- 沉浸式首屏下方可放搜索入口和 2-3 个可信业务指标，如本周活动数、撮合需求数、机构会员数。

### 常规顶部品牌区

- 非沉浸式一级页面可继续使用深蓝品牌区。
- 背景：`brand` 到 `brand-light` 的低调线性渐变，或纯 `brand`。
- 标题白色，说明使用 `rgba(255,255,255,0.7)`。
- 可承载 2-3 个关键指标，但不要超过一行。

### 卡片

- 背景 `#FFFFFF`。
- 阴影：`0 2px 16px rgba(10,31,92,0.10)`。
- 卡片不再嵌套卡片；列表项可使用分割线。
- 点击反馈只做透明度或轻微按压，避免悬浮漂移动效。

### 按钮

- 主按钮：深蓝渐变或纯深蓝，白字。
- 高价值按钮：金色，用于会员、报名、特价、优先动作。
- 次级按钮：白底深蓝描边。
- 小按钮高度约 30-34px，常规按钮高度约 40-44px。

### 标签

- 信息标签：浅蓝底 + tech 蓝字。
- 权益/会员标签：浅金底 + gold 字。
- 成功状态：浅绿底 + success 字。
- 紧急/价格：红色只用于少量状态，不作为主品牌色。

### 图标

- 正式实现使用 SVG 或组件图标，不使用 emoji。
- 图标容器 32-46px，圆角 8-14px。
- 同一页面图标线宽、尺寸和颜色必须一致。

## 页面方向

### 首页

- 首屏突出商业撮合、线下活动、资源对接和 2-3 个可信数据。
- 首屏可使用真实背景图，推荐题材：活动现场、商务会议、资源对接会、路演现场、企业洽谈。
- 首页标题文案应直接指向业务，例如“本周企业资源对接会”“财税机构增长沙龙”，不要写成书籍阅读或文化体验。
- 快捷入口控制在 4-6 个，文字短而业务明确。
- 平台动态使用列表，不做过多卡片堆叠。

### 服务商城

- 使用纵向列表卡片，突出服务名称、简短说明、明确动作。
- 价格和促销信息需要克制，避免整页红色。

### 行商书苑

- 作为品牌入口时，用于承载平台介绍、伙伴权益、活动体系、撮合服务标准和会员规则。
- 更像“商业服务说明书”，不做书籍阅读、传统书院或文化散文式表达。

### 我的

- 顶部强调认证身份和机构信息。
- 会员权益卡可用金色，但内容要简洁。
- 订单、课程、商机等入口保持 3 列以内。

## 反 AI 味约束

- 不使用大面积紫蓝渐变、发光球、玻璃拟态堆叠。
- 不使用书籍、古风书院、山水文旅作为默认业务主视觉，除非具体活动本身就是相关主题。
- 不使用 emoji 图标填充界面。
- 不写空泛文案，如“开启智慧未来”“赋能无限可能”。
- 不把每个模块都做成独立浮卡；列表、分割线和信息层级更自然。
- 不使用过度圆角，卡片保持 12px 左右。
- 数据必须像业务指标，不像随机生成的口号。

## 可访问性与动效

- 正文与背景对比度至少 4.5:1。
- 页面动效控制在 150-250ms。
- 避免循环装饰动画。
- 沉浸式导航的滚动切换应轻量，避免复杂视差、长时间模糊动画或影响滚动性能的背景动画。
- 移动端可点击区域不小于 44px，背景图切换、搜索、胶囊模拟按钮等都需满足触达尺寸。
- 使用固定尺寸的入口、卡片缩略区和按钮，减少内容加载时跳动。
- 尊重 `prefers-reduced-motion`。
