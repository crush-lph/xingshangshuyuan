# 原型页面还原映射表

## 状态枚举

- `已实现`：页面结构、真实数据、主操作和状态反馈已满足当前阶段验收。
- `部分实现`：已有页面或部分模块，但仍缺少关键字段、状态或交互。
- `接口缺失`：页面需要的核心接口或字段当前不可用。
- `暂缓`：当前阶段不处理，但后续阶段会处理。
- `不在范围内`：不属于微信小程序端或已明确不做。

## Phase 5 组件与视觉收口

- `PageShell` 已补内容区标题、副标题和可选 eyebrow；强视觉头图页面可通过 `showHeader={false}` 避免标题重复。
- 页面级空态优先使用 `StateNotice`，旧 `EmptyState` 保留为组件兼容但不再作为本阶段触达页面的页面级状态。
- 后台页面已通过负向文案扫描，不展示假票号、假核销、假审核、假确认、假分配或假撮合结果。
- 动态 class 扫描只允许枚举映射类，不新增 Tailwind 无法扫描的运行时颜色拼接。

## Phase 6 票务与核销闭环

- 电子票页携带 `registration_id` 调用 `/api/event/ticket`，展示后台返回的电子票二维码、二维码原文、参会人和状态。
- 活动报名提交后不再丢弃响应，优先把 `registration_id` 传入电子票页；付费活动或历史入口未返回报名记录时保留活动详情兜底说明。
- 后台核销页调起微信扫一扫，扫码后把二维码内容作为 `code` 提交到 `/api/event/checkin`，并展示后台返回的报名记录、参会人、手机号和状态。
